# llm-wiki: 个人 AI 知识库系统

## 核心理念

**"把碎片化信息变成持续积累、互相链接的知识库"** —— 把 AI 当编译器，一次性把素材编译成结构化百科，而不是每次查询都重新翻找。

## 目录约定

```
ai-wiki/
├── raw/                    # 原始素材，按来源分类
│   ├── webpage/
│   ├── x/                  # Twitter/X
│   ├── wechat/             # 公众号
│   ├── xiaohongshu/
│   ├── zhihu/
│   ├── youtube/            # 视频转录文本
│   ├── pdf/
│   └── local/              # 本地 .md / .txt
└── wiki/                   # 整理产出
    ├── entities/           # 实体页（人/概念/工具/项目）
    ├── topics/             # 主题页（聚合多个实体）
    ├── sources/            # 来源摘要（每篇素材一份）
    └── index.md            # 总入口
```

## 工作流路由器

根据用户意图分发到对应 workflow：

| 意图关键词 | 工作流 | 状态 |
|-----------|--------|------|
| "初始化"、"建库" | **init** | TODO |
| "采集"、"消化"、"加这一篇" | **ingest** | TODO |
| "批量"、"全部消化" | **batch-ingest** | TODO |
| "查一下"、"问知识库" | **query** | TODO |
| "综合报告"、"深度分析" | **digest** | TODO |
| "检查"、"lint"、"健康度" | **lint** | TODO |
| "状态"、"统计" | **status** | TODO |
| "图谱"、"关系图" | **graph** | TODO |

每个 workflow 的详细步骤待填充到 `workflows/<name>.md`。

## 处理模式

- **完整处理**（>1000 字）：抽取实体、生成摘要、写实体页、链接修复
- **简化处理**（≤1000 字）：直接归入来源摘要，标记关键实体不单独建页

## 质量约束（强制）

完成任何 ingest/batch-ingest 后必须满足：

- [ ] 文件名 = `[[wikilink]]` 内的名字（无大小写/空格不一致）
- [ ] 实体页 ≥ 1500 字，无占位符（"待补充"/"TODO"）
- [ ] 每篇 source 摘要包含 ≥ 2 段原文摘录与"实践内容"段（代码/prompt/教程保留原样）
- [ ] frontmatter `sources: []` 字段非空
- [ ] 主题页 ≥ 5 个核心要点
- [ ] 出现 3+ 可对比实体时自动建对比页；出现 5+ 来源同主题时自动建综合页

## 自动联动

每次 ingest/batch-ingest **结束前**强制调用 `lint` workflow 做一次健康检查，输出问题清单后再决定是否完成。

## 多语言

支持 `zh` / `en`，模板按语言分文件存放。默认 `zh`。
