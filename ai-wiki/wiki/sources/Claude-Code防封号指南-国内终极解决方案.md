# Claude Code 防封号指南：国内终极解决方案

## 一句话概括

针对 Anthropic 引入 Persona 第三方身份核验（KYC）导致大量国内用户被封号的问题，从网络环境、行为画像、Persona 认证底层逻辑三个维度给出系统性规避方案，核心思路是维持统一的"在美国出差的正常人类"画像。

## 实践内容

### Persona 是什么

Persona 是旧金山的第三方身份核验服务商，通过多维度 AI 决策引擎判断用户身份。主要客户为金融、加密、医疗等对身份验证要求严格的行业。Anthropic 将 Claude 的用户验证提升到了"跟开银行账户差不多的级别"。

### 封号原因

Claude 不允许中国大陆地区（含港澳）使用。关键区分：Claude 禁止的是**地区访问**，而非禁止中国人使用。被外派到美国工作的中国人在美国本地网络访问 Claude 是合理合规的。

核心要求两层：
1. 网络环境确实在可用区域
2. 是真实人类在使用

### 网络环境配置

**不推荐：**
- Express、Nord 等传统 VPN
- 廉价机场

原因：IP 地址池已被风控系统标记，且存在严重的超售导致 IP 频繁跳变。

**推荐方案：**
- 使用主打 BGP/IEPL 专线的高质量机场
- IEPL 内网专线防止断流，保持对外 IP 固定
- 较高定价门槛天然隔离大量并发自动化脚本和高风险用户
- 原生支持 Clash、Surge 等客户端，方便路由分流和防止 DNS 泄露

### 行为画像规范

- 循序渐进地使用，不要一上来就买最高订阅
- 遵循人类作息，避免 7x24 小时高频调用
- 时区对齐：如选纽约节点，系统时区也应设为纽约

### Persona 认证底层逻辑（三层验证）

**第一层：证件真实性验证**
- OCR 识别
- 防伪特征检测
- 数据库比对

**第二层：人证一致性验证**
- 活体检测
- 人脸比对

**第三层：风险信号综合评估**
- 收集 IP、设备指纹、注册行为模式、地理位置一致性等信号
- 生成风险评分
- 可能通过 WebRTC 穿透代理读取真实本地 IP

目前 Persona 不会普发，仅在触碰高危功能或异常行为时触发。

### Persona 认证注意事项

核心宗旨：证明自己是一个出差在美国工作的正常人类。

**操作步骤：**

1. **上传证件**：使用护照，中国身份证不可用
2. **验证环境**：使用指纹浏览器（如 AdsPower 或 HubStudio），创建无痕环境，务必禁用 WebRTC
3. **手机扫码活体验证**：
   - 必须拔掉手机 SIM 卡（防止基站定位和运营商暴露）
   - 关闭 GPS
   - 手机连接 WiFi 并使用与电脑端完全相同的代理节点
   - 全程保持网络畅通

### 额外加分项

- 使用有历史记录的 Gmail 老账号注册
- 支付走苹果 App Store 礼品卡内购

### 总结

核心逻辑：保持统一的"在美国出差的中国人"画像，让所有网络伪装、路由配置、使用习惯都指向这一结论。
